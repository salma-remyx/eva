# WER Normalizer Config Reference

Each `<lang>.json` file drives the spoken-numberв†’digit normalizer used by the
`stt_wer` metric. Files are auto-generated by `add_culture_data.py` and can be
edited by hand when the LLM output is incorrect.

> **Quick diagnosis**: delete the file and re-run `add_culture_data.py` to
> regenerate from scratch, or edit individual fields below. The normalizer
> reloads the file on every call, so changes take effect immediately.

---

## Pipeline overview

Text passes through two layers:

1. **Outer (text) normalizer** вЂ” lowercases, removes punctuation, strips filler
   words, expands abbreviations, handles thousand/decimal separators.
2. **Inner (number) normalizer** вЂ” a left-to-right state machine that converts
   number word sequences to digit strings.

The fields below control both layers.

---

## Vocabulary tables

These are the fields most likely to need manual correction.

> **One-bucket rule**: a word must appear in exactly **one** of `cardinals`,
> `tens`, or `scaling_units`. The engine checks them in that order; a word in
> two buckets is always treated as a cardinal and may break composition.

### `cardinals` `{word: int}`

Fixed-value cardinal words that are **not** multiples of 10.

- For most languages: 1вЂ“19.
- Also includes lexicalized larger values that aren't multiples of 10 вЂ” e.g. a
  single-token word for 500 in languages where "five-hundred" is one word (put
  it here, **not** in `scaling_units`).
- For `lexicalized_below_100` languages (Hindi native numerals etc.): 1вЂ“99.

```json
"cardinals": {"one": 1, "two": 2, ..., "nineteen": 19}
```

Common mistake: LLM puts "twenty" / "vingt" / "zwanzig" here. These are
multiples of 10 and belong in `tens`.

### `cardinal_variants` `{word: int}`

Alternate surface forms of an already-listed cardinal вЂ” gender/case/elision
variants that map to the same integer.

```json
"cardinal_variants": {"une": 1, "un": 1}   // French: feminine "une" = 1
```

The engine merges these into the same lookup as `cardinals` at startup.

### `cardinals_suffixed` `{word: [int, suffix]}`

Ordinal and similar forms where the output should be `digit + suffix`.

```json
"cardinals_suffixed": {
  "first":    [1, "st"],
  "second":   [2, "nd"],
  "deuxiГЁme": [2, "ГЁme"],
  "premiГЁre": [1, "ГЁre"]
}
```

Diacritics in both the key and the suffix are stripped at runtime, so you can
write `"deuxiГЁme": [2, "ГЁme"]` and it will match `"deuxieme"` in the
normalized text and emit `"2eme"`.

### `tens` `{word: int}`

Multiples of 10 from 20 to 90 (or as many as the language has compositionally
вЂ” French stops at 60 because 70вЂ“99 are vigesimal).

Include **all surface forms**, especially plurals:

```json
"tens": {"vingt": 20, "vingts": 20, "trente": 30, ...}
```

For `lexicalized_below_100` languages (Hindi native), leave this empty `{}`.

### `tens_suffixed` `{word: [int, suffix]}`

Same shape as `cardinals_suffixed`, for tens-ordinals (rare вЂ” needed only if a
language has a distinct ordinal form for "twentieth", "thirtieth", etc.).

### `scaling_units` `{word: int}`

**True multiplicative** scaling words: hundred, thousand, million, billion.
A word goes here **only** when it scales a preceding number:

```json
"scaling_units": {
  "hundred": 100, "thousand": 1000,
  "million": 1000000, "billion": 1000000000,
  "lakh": 100000, "crore": 10000000
}
```

**Do not** put a fixed-value compound like "five-hundred-as-one-word" here вЂ”
that belongs in `cardinals`. Test: can the word be preceded by a different
number to scale it (e.g. "two thousand", "three hundred")? If yes в†’ here. If
no в†’ `cardinals`.

### `scaling_units_suffixed` `{word: [int, suffix]}`

Same shape as `cardinals_suffixed`, for suffixed scaling-unit ordinals (very
rare).

### `zeros` `[str]`

Words meaning zero. Each is treated as string-concatenation of "0" rather than
numeric addition.

```json
"zeros": ["zero", "oh", "nil"]
```

## Connectives

### `conjunction_word` `str | null`

The spoken word that links a large unit to a smaller one: English "and", French
"et", German "und". Set `null` if the language never uses one.

When set, the engine normally pauses number accumulation at the conjunction and
restarts. `conjunction_ignore_prev` controls exceptions.

### `conjunction_ignore_prev` `[str]`

Category names (exactly: `"scaling_units"` and/or `"tens"`) where a preceding
word of that category allows the conjunction to pass through without breaking
the number. Default `["scaling_units", "tens"]` is correct for most languages.

Example: English "one hundred **and** one" вЂ” "and" follows a scaling unit
(hundred) so it should not flush; this works because `"scaling_units"` is in
the list.

Leave `[]` only if the language's conjunction word is never used mid-number.

### `decimal_word` `str | null`

The spoken decimal separator ("point", "virgule", "komma", "punto", etc.).
After this word the engine emits a "." and resumes in fractional mode.
`null` if the language has no such word.

## Structural flags

### `reversed_units` `bool`

`true` for languages where the units word precedes the tens word, optionally
glued by the `conjunction_word`:

- German: "einundzwanzig" (one-and-twenty = 21)
- Dutch: "eenentwintig"
- Arabic: "wahid wa-Кїishrun" (one and twenty)
- Hebrew: "echad ve-esrim"

The preprocessor splits glued compounds and swaps unit/tens order before the
state machine runs.

### `split_hyphenated_numbers` `bool`

`true` if the language writes multi-word numbers with hyphens that must be
split into separate tokens:

- French: "quatre-vingt-dix" в†’ ["quatre", "vingt", "dix"]
- Spanish: "veinte-y-uno" (if written with hyphens)

### `vigesimal_trigger_words` `[str]`

Words that participate in vigesimal (base-20) multiplication. Used for
French-family systems where "quatre-vingts" = 4 Г— 20 = 80.

Include all surface forms: `["vingt", "vingts"]`.

Leave `[]` for all other languages.

### `vigesimal_residuals` `[int]`

The `value % 100` values that are allowed to multiply a vigesimal trigger word.
French: `[4, 5, 6, 7, 8, 9]` вЂ” only quatre/cinq/.../neuf Г— vingt make sense.

### `additive_teen_residuals` `[int]`

The `value % 100` values where a following teen (cardinal в‰Ґ 10) should be
**added** to the running total rather than string-concatenated. `0` is always
implicit.

French needs `[0, 60, 80]`:
- value = 60 ("soixante") + "dix" (10) в†’ add в†’ 70
- value = 80 ("quatre-vingts") + "dix" (10) в†’ add в†’ 90

Most languages: leave as `[0]` (default).

## Text-level fields

### `ignore_patterns` `str`

A single Python regex (or empty string) that matches filler/disfluency words to
remove before comparison. Generated from the filler word list.

```json
"ignore_patterns": "\\b(uh|um|hmm|er|ah)\\b"
```

Test it with `re.findall(pattern, text)`. If the pattern is invalid the
normalizer silently ignores it.

### `replacers` `{regex: replacement}`

Map of `{pattern: expansion}` for abbreviation normalization. Each key is a
Python regex; the value is the expanded form.

```json
"replacers": {
  "\\bdr\\b": "doctor",
  "\\bprof\\b": "professor"
}
```

Patterns are applied before number normalization. Invalid patterns are silently
dropped.

### `thousand_separators` `[str]`

Which punctuation characters act as thousand separators and should be removed
from digit strings before comparison. Each element is one of: `"comma"`,
`"dot"`, `"space"`.

```json
"thousand_separators": ["dot", "space"]   // German: 1.000 and 1 000
```

### `decimal_separator` `"dot" | "comma"`

Whether the decimal point in digit form is `.` or `,`.

- `"dot"`: English, Hindi, Arabic вЂ” `3.14`
- `"comma"`: French, German, Spanish, Polish вЂ” `3,14`

Both sides of the WER comparison are normalized to a canonical `.` internally.

### `spelling_map_path` `str | null`

Filename (relative to this directory) of a `{variant: canonical}` JSON map for
regional spelling normalization. Used for languages with significant orthographic
divergence between dialects (e.g. `en_spelling.json` for Britishв†’American).
`null` for most languages.

### `preserve_combining_marks` `bool`

`true` for scripts where Unicode combining marks are phonemically essential:
Devanagari, Bengali, Tamil, Arabic, Hebrew, Thai, and related scripts.

When `true`, the outer normalizer uses NFC (no decomposition) and only strips
punctuation/symbols, leaving all letter and combining-mark characters intact.
When `false` (default), diacritics are stripped for comparison.

## Rarely-edited fields

These are almost never wrong but are documented for completeness.

| Field | Default | Notes |
|---|---|---|
| `letter_class` | `"a-z"` | Regex character class for letter/digit boundary detection. Set from locale defaults by script family. |
| `ordinal_suffix_pattern` | `""` | Regex for ordinal suffixes to strip from digit-ordinal tokens during comparison. |
| `half_pattern` | `null` | Regex matching "and a half" or equivalent. |
| `half_replacement` | `null` | Replacement string for `half_pattern` (e.g. `"point five"`). |
| `one_word` | `null` | If set, postprocesses the digit `"1"` back to a word (rare). |
| `one_plural_suffix` | `""` | Suffix added to `one_word` when plural applies. |
| `cents_connector` | `null` | Word optionally spoken between a currency amount and its cents. |
| `strip_space_before_apostrophe` | `false` | Removes space before `'` (French contractions). |
| `cardinal_continuation_on_prev_cardinal` | `false` | Enables English "one oh one" (101) form where a cardinal follows another cardinal directly. |

## Worked examples

### English "one hundred and one" в†’ `101`

| Token | Match | State |
|---|---|---|
| one | cardinals[one]=1 | value=1 |
| hundred | scaling_units[hundred]=100 | value=1Г—100=100 |
| and | conjunction (hundred in scaling_units в†’ passes through) | value=100 |
| one | cardinals[one]=1, value%10==0 в†’ add | value=101 |

### French "soixante-dix-neuf" в†’ `79`

Preprocessor splits on hyphens: ["soixante", "dix", "neuf"]

| Token | Match | State |
|---|---|---|
| soixante | tens[soixante]=60 | value=60 |
| dix | cardinals[dix]=10, nв‰Ґ10, value%100=60 в€€ additive_teen_residuals в†’ add | value=70 |
| neuf | cardinals[neuf]=9, value%10в‰ 0 в†’ string-concat | value="79" |

### French "quatre-vingt-dix-neuf" в†’ `99`

Preprocessor splits: ["quatre", "vingt", "dix", "neuf"]

| Token | Match | State |
|---|---|---|
| quatre | cardinals[quatre]=4 | value=4 |
| vingt | tens[vingt]=20, vigesimal trigger: 4%100=4 в€€ vigesimal_residuals в†’ value=0+(4Г—20) | value=80 |
| dix | cardinals[dix]=10, nв‰Ґ10, value%100=80 в€€ additive_teen_residuals в†’ add | value=90 |
| neuf | cardinals[neuf]=9, value%10==0 в†’ add | value=99 |

### German "einundzwanzig" в†’ `21`

`reversed_units=true`, `conjunction_word="und"`. Preprocessor splits the glued
compound: "ein und zwanzig". Then swaps unit before tens: "zwanzig ein".

| Token | Match | State |
|---|---|---|
| zwanzig | tens[zwanzig]=20 | value=20 |
| ein | cardinals[ein]=1, value%10==0 в†’ add | value=21 |
